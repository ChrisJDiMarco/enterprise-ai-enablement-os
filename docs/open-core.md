# Open-Core Model

Enterprise AI Enablement OS is intended to be useful as an open-source product and credible as the foundation for a hosted enterprise platform.

## Open-Source Core

The public core should include the operating model a company needs to begin managing AI responsibly:

- AI portfolio intake and prioritization
- work signal capture
- use case factory
- governed AI Skill records
- workflow design
- model and tool governance concepts
- evals, traces, controls, and approval evidence
- proof ledger
- ROI and adoption reporting
- AI assistant command hub
- local development and deployment documentation

This gives practitioners, consultants, developers, and AI transformation leaders a real system they can inspect, run, extend, and trust.

## Hosted Enterprise Platform

A paid hosted version can focus on operational depth that companies do not want to build and maintain themselves:

- managed cloud hosting, upgrades, backups, and observability
- SSO, SCIM, audit exports, and advanced tenant administration
- managed connector broker for Slack, Microsoft 365, Jira, ServiceNow, SharePoint, Workday, Google Workspace, and internal systems
- enterprise secret storage and key rotation workflows
- advanced policy packs for regulated teams
- executive reporting automation and scheduled board packets
- deployment support, onboarding, enablement, and implementation services
- premium support, SLAs, and customer-specific compliance evidence

## Boundary Principle

The open-source product should not feel like a hollow demo. The hosted version should win because it is easier, safer, better supported, and more operationally complete for large organizations.
